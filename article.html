CSS3: There's a mixin for that

Creating a library of mixins for CSS3 using SASS

Posted on: Nov 13, 2011

This is a very simple concept, but one that has saved me a lot of time and the frustration of looking up the correct syntax and vendor selectors of CSS3 declarations.

For all those who are not yet using a CSS precompiler, such as SASS, or LESS, it's about time that you gave it a shot. Your efforts will be rewarded. For all of those who are, I will be using SCSS (a flavor of SASS).

I have been adding to my library of mixins for about a year now, and the best part is that I am continually building something useful for all future projects.

Mixins are little chunks of programmatic CSS that can be called inside of any class and passed custom values, so you can imagine how useful they might be when creating effects that have various options and magnitudes. Here's an example of one that I use all the time.

@mixin rounded_corners( $magnitude ) {
    -moz-border-radius: $magnitude;
    -webkit-border-radius: $magnitude;
    border-radius: $magnitude;
    -khtml-border-radius $magnitude;
}

And just like that, we have a pretty lil' rounded corner function that can be called with one line. 

.class_name {
    @include rounded_corners( 5px );
}

Hot. We could even take it a little further, and make another mixin to round a specific corner.

@mixin rounded_corner( $magnitude, $x, $y ) {
    -moz-border-radius-#{$y}#{$x}: $magnitude;
    -webkit-border-#{$y}#{$x}: $magnitude;
    border-#{y}-#{$x}: $magnitude;
}

Hopefully you're starting to see the power of this technique. I could just as easily make mixins for gradients, drop shadows, or animations to my hearts content.
