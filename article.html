CSS3: There's a mixin for that

Creating a library of mixins for CSS3

by Jordan Cooperman

Posted on: Nov 13, 2011

Mixins are small, programmatic chunks of CSS used by CSS precompilers such as SASS and LESS. In SCSS (a flavor of SASS), they look like this: 

@mixin inline-block( $vertical-align: middle ) {
    display: inline-block;
    *display: inline;
    *zoom: 1;
    vertical-align: $vertical-align;
}

Mixins can be called as functions by CSS selectors and passed optional arguments: 

.class_name {
    @include inline-block( middle );
}

Once compiled, the above mixin would look like this: 

.class_name {
    display: inline-block;
    *display: inline;
    *zoom: 1;
    vertical-align: middle;
}

Because we are precompiling our CSS, mixins are not rendered into CSS unless they are explicitely called by a selector. This means that we can create an extensive library of mixins without paying for it with performance.

Over the last year or so, I have been building a library of mixins with SASS to do a variety of tasks, and one of the most valuable tasks that I have found is with regards to CSS3.

CSS3 is not supported by all browsers, and as a result, browser vendors often use custom syntaxes for CSS3 property names that reflects the browser's name and which are ignored by all other browsers as invalid CSS. This allows vendors to experiment with new techniques without worrying about how the CSS will validate in other browsers.

While this is a great solution for evolving browsers, it unfortunately means that we are often faced with several different syntaxes for the same CSS3 property in different browsers. So let's aleviate that pain using mixins.

To start, let's use an SCSS mixin to create rounded corners: 

@mixin rounded_corners( $magnitude ) {
    -moz-border-radius: $magnitude;
    -webkit-border-radius: $magnitude;
    border-radius: $magnitude;
    -khtml-border-radius $magnitude;
}

And just like that, we have a pretty lil' rounded corners function that can be called with one line of code and will work in all modern browsers that support rounded corners. Let's use it: 

.class_name {
    @include rounded_corners( 5px );
}

Hot. We could take it even further and make another mixin to round a specific corner:

@mixin rounded_corner( $magnitude, $x, $y ) {
    -moz-border-radius-#{$y}#{$x}: $magnitude;
    -webkit-border-#{$y}#{$x}: $magnitude;
    border-#{y}-#{$x}: $magnitude;
}

Hopefully you're starting to see the power of this technique. We could just as easily make mixins for gradients, drop shadows, or animations. Let's make one for CSS3 transitions now: 

@mixin transition( $property, $time, $effect: ease-in-out ) {
    -webkit-transition: $property $time $effect;
    -moz-transition: $property $time $effect;
    -o-transition: $property $time $effect;
    -ms-transition: $property $time $effect;
    transition: $property $time $effect;
}

That's just marvelous, let's use it: 

a {
    color: #333;
    @include transition( color, 0.2s );

    &:hover {
        color: #ff0000;
    }
}

Brilliant, now we've got a color transition on hover with one line of code.

Hot damn.
