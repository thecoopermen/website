CSS3: There's a mixin for that

Creating a library of mixins for CSS3 using SASS

Posted on: Nov 13, 2011

Mixins are small, programmatic chunks of CSS used by CSS precompilers such as SASS and LESS that can be called as functions by CSS selectors and passed optional arguments.

Because we are precompiling our CSS, mixins are not rendered into CSS unless they are explicitely called by a selector. This means that we can create an extensive library of mixins without paying for it with performance.

Over the last year or so, I have been building a library of mixins to do a variety of tasks, and one of the most valuable tasks that I have found is with regards to CSS3.

CSS3 is not supported by all browsers, and as a result, browser vendors often use a custom syntax for CSS3 property names that reflects the browser's name and which will be ignored by all other browsers as invalid CSS. This allows vendors to experiment with new techniques without worrying about how the CSS will validate in all browsers.

Unfortunately for us, this means that in many situations, we are faced with several different syntaxes to accomplish the same goal in multiple browsers. Here enters the mixin.

Here's an example of how we may us mixins to aleviate the pain of multiple vendor selectors for CSS3 rounded corners: 

@mixin rounded_corners( $magnitude ) {
    -moz-border-radius: $magnitude;
    -webkit-border-radius: $magnitude;
    border-radius: $magnitude;
    -khtml-border-radius $magnitude;
}

And just like that, we have a pretty lil' rounded corners function that can be called with one line of code and will work in all modern browsers that support rounded corners. Let's use it: 

.class_name {
    @include rounded_corners( 5px );
}

Hot. We could take it even further and make another mixin to round a specific corner:

@mixin rounded_corner( $magnitude, $x, $y ) {
    -moz-border-radius-#{$y}#{$x}: $magnitude;
    -webkit-border-#{$y}#{$x}: $magnitude;
    border-#{y}-#{$x}: $magnitude;
}

Hopefully you're starting to see the power of this technique. I could just as easily make mixins for gradients, drop shadows, or animations. Let's make one for CSS3 transitions now: 

@mixin transition( $property, $time: 0.25s, $effect: ease-in-out ) {
    -webkit-transition: $property $time $effect;
    -moz-transition: $property $time $effect;
    -o-transition: $property $time $effect;
    -ms-transition: $property $time $effect;
    transition: $property $time $effect;
}

That's just marvelous, let's use it: 

a {
    color: #333;
    @include transition( color, 0.2s );

    &:hover {
        color: #ff0000;
    }
}

Brilliant, now we've got a color transition on hover with one line of code.
